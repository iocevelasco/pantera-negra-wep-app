# Documentación del Proyecto Pantera Negra

Esta carpeta contiene toda la documentación importante para la interpretación del código del proyecto.

## 📚 Índice de Documentación

### Documentación Principal

- **[README.md](./README.md)** - Documentación principal del proyecto, incluyendo estructura, instalación, configuración y guías de uso.

### Configuración y Setup

- **[RECAPTCHA_SETUP.md](./RECAPTCHA_SETUP.md)** - Guía completa para configurar reCAPTCHA v3 en el proyecto, incluyendo solución de problemas comunes.

- **[RECAPTCHA_TROUBLESHOOTING.md](./RECAPTCHA_TROUBLESHOOTING.md)** - Guía detallada para solucionar problemas comunes de reCAPTCHA, especialmente el error "hostname: undefined".

- **[API_CONFIG_README.md](./API_CONFIG_README.md)** - Documentación sobre la configuración de la API, incluyendo base de datos y variables de entorno.

- **[API_CONFIGURATION.md](./API_CONFIGURATION.md)** - Guía del sistema de configuración centralizada de la API.

- **[I18N_README.md](./I18N_README.md)** - Documentación sobre el sistema de internacionalización (i18n) del proyecto.

### Análisis y Recomendaciones

- **[REGISTRO_USUARIO_ANALISIS.md](./REGISTRO_USUARIO_ANALISIS.md)** - Análisis detallado del flujo de registro de usuarios.

- **[ATTENDANCE_CHECKIN_RECOMMENDATIONS.md](./ATTENDANCE_CHECKIN_RECOMMENDATIONS.md)** - Recomendaciones y análisis sobre el sistema de check-in de asistencia.

### Características

- **[FEATURES.md](./FEATURES.md)** - Lista de características y funcionalidades del proyecto.

## 📖 Cómo Usar Esta Documentación

1. **Para desarrolladores nuevos**: Empieza con [README.md](./README.md) para entender la estructura del proyecto.

2. **Para configurar servicios externos**: Consulta [RECAPTCHA_SETUP.md](./RECAPTCHA_SETUP.md) para reCAPTCHA.

3. **Para entender flujos específicos**: Revisa los documentos de análisis según tu necesidad.

4. **Para configurar la API**: Consulta [API_CONFIG_README.md](./API_CONFIG_README.md).

## 🔄 Actualización de Documentación

Cuando agregues nueva documentación:
1. Colócala en esta carpeta (`packages/shared/docs/`)
2. Actualiza este índice
3. Asegúrate de que el nombre del archivo sea descriptivo

## 📝 Notas

- Todos los archivos de documentación están en formato Markdown (.md)
- La documentación está centralizada aquí para facilitar su mantenimiento y acceso
- Esta carpeta es parte del paquete `shared`, por lo que está disponible en todo el monorepo
